# Builder Philosophy

- Prefer open, modular, forkable systems.
- Prefer low-friction tools that ordinary people can adopt.
- Use frontier tools to repair old systems, not just invent novelty.
- Avoid dependence on any single AI provider.
- Multi-provider routing is a feature.
- Local fallback matters, even if slower.
- Agent wrapping matters more than the UI shell.
- Wallet actions should be policy-gated, not raw and careless.
- Payment rails that let agents/tools charge natively may matter more than chain fandom.
- Real value likely lives in workflow design, trust, permissions, memory, portability, and usability.
- Practical utility beats tribal allegiance.
