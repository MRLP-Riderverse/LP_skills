# Directory view controls — resolved pattern

Use this when refining the public directory browser:

- **Show all is the default view**. The first impression should not force the user to opt in before seeing entries.
- Keep **Clear** as the reset action.
- Remove redundant **A–Z** and **Category** buttons when the dropdown filters already expose those dimensions.
- Keep the control surface compact: search + mood dropdown + category dropdown + city dropdown is enough for the default browse state.
- If the user later asks for sorting again, reintroduce it only when it adds new information, not as a duplicate of the dropdown filters.

This pattern came out of the directory browse refinement session where the user wanted a calmer initial view with fewer duplicated controls.
