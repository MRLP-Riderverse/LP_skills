# Relation graph lookup notes

Session takeaway: directory lookup should eventually surface related and nearby places to make the cards more useful for casual discovery and event planning.

Helpful relationship buckets for clean entries and lookup UX:
- related places
- nearby places
- shared corridor / district
- shared events
- optional backlinks between linked entries

Observed Bathurst pattern from this session:
- Main St / downtown nightlife-food strip: NB Distillery, La Louga, Bootlegger
- Waterfront brewery corridor: 13 Barrels, Four Rivers

Desired lookup behavior later:
- when a user asks for a card, show the match plus a few related/nearby places
- if appropriate, use the related places to suggest a small cluster for outings or event planning
- keep the discovery friendly and lightweight, not overwhelming
