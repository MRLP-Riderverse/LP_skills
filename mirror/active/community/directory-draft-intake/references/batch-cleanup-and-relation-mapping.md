# Batch cleanup and relation mapping notes

Session-derived cues for alphabetic inbox polish passes.

## Observed batch rhythm
- Work in slices: A-C, D-H, I-O, P-Z.
- Clean each slice, sanity-check it, then commit that slice before moving on.
- Export once the full requested batch is done.

## Default cleanup moves
- Add `Category` and `Tags` when the type is obvious from the notes.
- Split long raw notes into short readable blocks.
- Add `Related places` when a listing clearly belongs to a corridor, plaza, shared site, or venue cluster.
- Preserve the user’s raw wording when it carries local meaning.
- Keep uncertain facts in notes or brief admin notes instead of forcing a guess.

## Relation hints that surfaced repeatedly
- Mall branch ↔ marina branch
- Dine-in ↔ nearby entertainment venue
- Campus ↔ other campus / satellite site
- Hotel ↔ airport / event / corridor businesses
- Waterfront / promenade / main-street clusters
- Shared gas-station / convenience clusters
- Parallel franchise branches in nearby towns should stay separate and be disambiguated by town or street name.

## Numericals
- Leading-number names are normal titles, not special exceptions.
- Keep names like `13 Barrels Brewing` intact.
- They belong in the same cleanup pass as the alphabetic slice that would otherwise contain them by sorting order.
