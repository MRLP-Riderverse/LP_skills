# Public research and formatting notes

Session-learned pattern for directory intake:

## Venue / location research sequence
Prefer the most authoritative public source that names the place and address, then enrich with secondary listings only as needed.

Typical order:
1. official mall / venue / business page
2. official social page (Facebook / Instagram) when the business uses it as its public face
3. event listing pages for live performance / venue context
4. map/listing pages for address confirmation

## What to preserve in the draft
- exact public-facing name if available
- mall / plaza / venue relationship
- street address
- business type signals, especially if the place is both a venue and a business location
- obvious public positioning phrases like "family style restaurant / bar / lounge"
- live-event / performance context when relevant

## Formatting pattern learned
For readable inbox drafts, break long note blobs into short lines or paragraphs.

Useful blocks:
- `Category:` for coarse type (`business`, `music`, `venue`)
- `Area:` for the region
- `Tags:` for search hints when the entry is likely to be queried by role or use case
- `## Notes` for the descriptive public summary
- `## Contact` when the entry has multiple social/contact handles
- `## Public source` with bullet URLs or `In person`
- `## Admin notes` with the default attribution

## Search helper tags
When the entry is a performer/artist, add a compact search hint such as:
- `Tags: music | artist`

When the entry is clearly a venue, use tags that help later discovery, such as:
- `Tags: venue | lounge | restaurant | bar`

Do not force these tags onto unrelated entries; they are helpful search rails, not schema law.
