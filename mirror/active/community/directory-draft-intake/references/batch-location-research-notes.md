# Batch location research notes

Session-derived workflow notes for drafting multiple local directory entries from public web research.

## Useful research order

1. Check for an existing draft in `inbox/` first.
2. Search official brand/location pages when available.
3. Use local search-engine result snippets and secondary listings to recover address / phone / hours when the official page is sparse.
4. Keep one draft per physical location.
5. If the brand has multiple branches in the same city or region, disambiguate the title with the street address or branch label.

## What worked well this session

- Franchise / chain brands often expose the strongest clue in the result title, not the page body.
- For branch lookup, titles and snippets were usually enough to draft a usable inbox card even when the page HTML was thin.
- When a place was already represented in the inbox, lookup before drafting prevented duplicates.
- Lightweight notes were enough during intake; cleanup can happen later.

## Drafting principle

Prefer a clearly named, imperfect draft over waiting for a perfect data set. If the public record is incomplete, preserve the best available public facts and keep the uncertainty in Notes.
