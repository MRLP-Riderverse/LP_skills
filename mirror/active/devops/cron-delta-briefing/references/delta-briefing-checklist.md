# Delta briefing checklist

Condensed operating notes for recurring cron briefings.

- Read the prior delivered briefing first.
- If available, inspect the most recent cron output artifact as the baseline.
- Compare against the last 3 briefings; repeated items are stale unless the topic changed.
- Skip quiet/dormant sources until they produce a new dated item.
- The final response must be the briefing itself or exactly `[SILENT]`.
- No follow-up note, no delivery confirmation, no table formatting for Telegram-bound output.
- Keep action items tied to the current cycle's new information only.
