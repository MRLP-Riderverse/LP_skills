# Creative Visual Demos

Use this skill for exploratory visual artifacts: quick HTML mockups, typographic experiments, generative sketches, and explanatory motion graphics.

## When to use
- User wants a disposable prototype before committing to implementation
- User wants several visual directions to compare side by side
- User wants an interactive sketch or creative-coding demo
- User wants an animation or motion graphic that explains an idea

## Core workflow
1. Pick the medium that best matches the question: HTML mockup, code sketch, or motion piece.
2. Keep the first version small and complete rather than elaborate and unfinished.
3. Verify the artifact visually in the medium it will be viewed in.
4. Use iteration to sharpen the idea, not to introduce unrelated variations.

## Subsections
### HTML sketches and mockups
Use when the user wants fast browser-based variants of a UI or page layout.

### Typographic and experimental layouts
Use when the output is more about visual composition than functional UI.

### Creative coding and generative sketches
Use when the user wants dynamic or algorithmic visuals.

### Motion and explanatory animation
Use when the output is a rendered animation or a mathematically-driven explainer.

## Quality bar
- The idea should be legible before the ornament lands
- The artifact should feel disposable, but not sloppy
- The medium should serve the concept instead of distracting from it
