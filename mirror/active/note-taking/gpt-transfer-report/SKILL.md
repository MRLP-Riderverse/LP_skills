# GPT Transfer Report

Bridge the GPT↔Hermes context gap. When the user pastes a session report or summary from ChatGPT/GPT, this skill turns it into a proper two-step process:

1. **Raw capture** — log the report to QuickThoughts unchanged, with the `Hermes · GPT Transfer` label
2. **Cross-reference + feedback** — compare against Hermes history and GBrain, then report back with intrigue and motivation

## When to Use

Load this skill when the user:
- Pastes a GPT session report, summary, or "notes from my GPT chat"
- Says something like "here's what GPT and I talked about" or "transferring from GPT"
- Shares a structured or semi-structured report clearly generated by another AI assistant

## Workflow

### Step 1: Raw Capture to QuickThoughts

Capture the report as-is to QuickThoughts using the `note` CLI:

```bash
NOTE_SOURCE_LABEL="Hermes · GPT Transfer" note "$(printf 'GPT Transfer [DATE] — [TITLE/SUMMARY LINE]\n\n[REPORT CONTENT]')"
```

Rules:
- Use `NOTE_SOURCE_LABEL="Hermes · GPT Transfer"` — this creates a visually distinct label in QuickThoughts so a human scanning the file can tell raw notes from Hermes captures from GPT transfers
- Preserve the report content as faithfully as possible — do not rewrite, compress, or interpret before capture
- Add a one-line prefix with date and a short title so it's searchable later
- If the report is very long, split into 2-3 notes with sequential labels like [1/3], [2/3], [3/3]
- Use the multiline format (prefix line + `-- |` continuation lines) for long content — see note-capture-workflow for syntax
- Do NOT skip the raw capture step even if the content seems familiar — the point is logging before analysis

### Step 2: Cross-Reference Against Hermes Context

After raw capture, search for overlaps and tensions:

1. **Session search** — use `session_search` with 3-5 key terms from the report to find prior Hermes conversations on the same topics
2. **GBrain search** — use `gbrain search` to check if concepts/decisions already exist in the knowledge base
3. **Memory scan** — check current memory entries for relevant prior stances or decisions

### Step 3: Analytical Response

Return a structured response with three sections:

**🆕 What's Genuinely New**
- Concepts, decisions, or insights that have NO prior coverage in Hermes sessions, GBrain, or memory
- These are the high-value signals — name them clearly
- If a concept is novel enough, offer to save it to memory

**♻️ What's Already Been Explored**
- Topics where Hermes and the user already went deep — cite specific sessions if possible
- Don't dismiss these as useless — they validate the direction and show consistency
- Note if the GPT report adds any new angle to an existing topic

**⚠️ Tensions or Contradictions**
- Any places where the GPT report's framing conflicts with concrete prior Hermes decisions
- Prior decisions take precedence unless the user explicitly wants to revisit them
- Flag these as "decide later" items, not auto-corrections

### Response Shape

```
**[Report Title] — Cross-Reference**

🆕 Genuinely new:
• [Concept X] — never appeared in Hermes context before
• [Insight Y] — new angle on existing topic

♻️ Already explored:
• [Topic A] — deep session on [date], [key decision]
• [Topic B] — daily brief territory for N+ weeks

⚠️ Tensions:
• [GPT says X] vs [Hermes decided Y on date] — prior decision holds unless you want to revisit
```

## Label Visual Distinction

Three label tiers in QuickThoughts for human legibility:
- **Raw user notes** → `⁜ HH:MM:SS | DD.MM.YY > [content]` *(no prefix)*
- **Hermes captures** → `⁜ HH:MM:SS | DD.MM.YY > Notes, by Hermes : [content]`
- **GPT transfers** → `⁜ HH:MM:SS | DD.MM.YY > Notes, by Hermes · GPT Transfer : [content]`

The `· GPT Transfer` suffix jumps out when scanning — same append-only protocol, same source of truth, just a legibility layer.

## Key Principles

- **Don't just say "saved"** — the whole point is feedback of intrigue and motivation
- **Raw capture first, analysis second** — never skip the QuickThoughts log
- **Prior decisions have weight** — don't let a fresh GPT session silently override a deliberate Hermes decision without flagging it
- **Celebrate the new stuff** — genuinely novel concepts deserve enthusiasm, not just clinical logging
- **Be concise but specific** — cite session IDs or dates when referencing prior coverage, don't hand-wave
- **Respect the federated cognition model** — GPT and Hermes are different contexts that should cross-pollinate, not merge into one truth

## Memory Interaction

- If a genuinely new concept surfaces that's *hot* (will matter across many future sessions), offer to save to memory
- If memory is near capacity, relocate older entries to QuickThoughts first (see memory offload policy)
- Do NOT auto-save to memory — always confirm with the user first
- **Every memory mutation (add/remove/replace) MUST be logged to QuickThoughts via `note` CLI** — log what changed, what was offloaded, what was added, and why. This is a reflex, not optional. Rationale: QuickThoughts is the human-readable audit trail; memory changes must be visible there.

## Source Attribution

When referencing prior Hermes context, attribute clearly:
- "From our May 23 session on Acadie.sol architecture..."
- "Your morning briefs have been tracking this since April..."
- "This was never discussed in Hermes before..."

This helps the user see the *difference* between GPT context and Hermes context — which is itself valuable signal.

## Multi-Report Sessions (Batch Transfers)

When the user pastes multiple GPT reports in one session (e.g. "here's the first one... and here's the second..."):

1. **Process each report individually** — raw capture + cross-reference per report, same as single-report workflow
2. **After the final report, deliver a meta-analysis** — compare all reports against each other AND against Hermes context. This is where the highest-value signal often lives. Look for:
   - **Convergent themes** — concepts that appear across multiple independent GPT sessions without coordination (these are the strongest signals — the pattern is finding the user, not the other way around)
   - **Complementary layers** — reports that naturally stack into what/how/why or other structural relationships
   - **Escalating reframes** — same identity expressed at different scales (e.g. "translator" as a job title → "translator" as a civilizational role)
3. **The meta-analysis is not optional for batch transfers** — it's the compounding step that turns isolated reports into a connected scaffold

Format for meta-analysis:

```
🔥 **Meta-across-all-reports:**
[Report 1] = [role in the stack — e.g. "what to build"]
[Report 2] = [role — e.g. "how to fund it"]
[Report 3] = [role — e.g. "why it matters"]
Recurring thread: [the pattern that showed up uninvited across all sessions]
```

### Pitfall: Don't let cosmic abstraction diminish concrete work

When a report operates at universe-scale (coordination eras, cosmic purpose, civilizational shifts), it can make concrete V1 work (directories, consulting sessions, static sites) feel small by comparison. It's not. The concrete work IS the infrastructure the big vision describes. Explicitly anchor: "This report is the why; the other reports are the how."

### User-originated insights within GPT transfer sessions

The user often extends GPT concepts with their own architectural ideas during the conversation (e.g. "the directory could be a queryable endpoint for agents"). These are NOT GPT insights — they're user innovations that surfaced while processing GPT content. Capture these to QuickThoughts separately with the standard `Hermes` label (not `GPT Transfer`), and attribute them clearly: "from user directly, not GPT."

## Examples

**Good response:**
> "🆕 Genuinely new: 'Federated cognition' as a named concept — never appeared in any Hermes session. ♻️ Already explored: Sovereign stack architecture has been daily brief territory for 5+ weeks. ⚠️ Tension: GPT pushes x402 payments early, but we explicitly chose Square-first for V1 on May 23."

**Bad response:**
> "Thanks for sharing! I've saved this to your notes." *(no analysis, no cross-reference, no value-add)*

**Good meta-analysis (batch):**
> "🔥 Meta-across-all-three-reports: Directory Protocol = what to build, AI Translator = how to fund it, Universe-Scale Systems = why it matters. Recurring thread: translator and steward. The Directory translates Acadians to each other. The consulting translates AI to businesses. The cosmic framework translates information to meaning."
