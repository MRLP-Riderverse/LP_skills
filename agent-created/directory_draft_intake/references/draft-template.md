# Directory Draft Template

Canonical shape for `inbox/_template.md` and notes-first draft capture.

```md
# Draft: <Name>

Category: <coarse type>
Area: <region>
Tags: <optional search hints>

## Description
<short plain-language description of the place>

## Notes
<additional readable context, vibe, and useful local details>

## Public data to carry forward
- Address: <...>
- Hours: <...>
- Phone: <...>
- Email: <...>
- Website: <...>

## Related places
- <nearby venue / sibling branch / corridor anchor>

## Admin notes
- Submitted by : Acadie.sol
- Telegram pass-through complete: [yes/no]
- Raw intake preserved: [yes/no]
- Restructured to template: [yes/no]
```

## Rules

- Only the name is mandatory at intake.
- Everything else can stay in `Notes` during raw capture.
- Preserve raw local wording instead of forcing early structure.
- Put reminders and stewardship notes in Admin notes.
- Keep `Submitted by : Acadie.sol` as the default attribution unless overridden.
- Do not force `In person` or any source label when the user did not explicitly say it.
- Never fail the draft because a field is missing.
- Do not pad with `not provided` placeholders at inbox stage.
- Before writing, check `inbox/` for a same-name or obvious twin entry.
- If a likely duplicate already exists, stop and report exactly: `I believe you already have an entry for this: <path>`.
- Use this template as the cleanup target after a raw capture lands.
