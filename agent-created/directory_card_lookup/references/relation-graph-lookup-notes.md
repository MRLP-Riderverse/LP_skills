# Relation Graph Lookup Notes

This note captures the intended future enhancement for Acadie.sol lookup cards:

- surface related places
- surface nearby places
- highlight shared corridors or districts
- surface shared events or event-planning clusters
- keep results human-readable first, with graph structure added later

Current practical cluster examples:
- Main St / downtown corridor: NB Distillery, La Louga, Bootlegger
- Waterfront corridor: 13 Barrels, Four Rivers

Suggested lookup UX:
- return the best direct match first
- add a small related/nearby section when available
- make random lookup outputs friendlier by showing adjacency and shared events

This is intentionally lightweight: a reference note for later implementation, not a schema spec.
