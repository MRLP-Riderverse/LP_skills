## Why this example matters

This event did not primarily live as a normal blog post. The useful facts were spread across:
- title + meta description
- schema-like date fields in raw HTML
- FAQ copy
- agenda subpage
- repeated visible session text
- one secondary recap for the actual announcement rollup

## Reusable lesson

When the official event page is a heavy marketing microsite:
1. confirm title, date, venue, and description from metadata
2. inspect agenda and FAQ subpages for scale and scope
3. pull a few representative named sessions from visible agenda text
4. use one secondary recap only after the primary facts are grounded
5. distinguish `officially scheduled` items from `reported announcements`
